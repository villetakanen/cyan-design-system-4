# Typography

The Design System uses `Lato` for headings and body text. The fonts should be imported from the `lato-font` package.

  <h2>Type Scale</h2>

The type scale is generated using https://typescale.com/ with "Major third" sacale and 1.25 ratio. The scale
is then adjusted to fit the the design system and units used in the design system (this might cause some _intentional_
rounding discrepancies between the scale and the actual font sizes used in the design system).

### Responsive Behavior

On mobile screens (620px and below), both HTML headings and their corresponding utility classes automatically scale down by one level to maintain readability and hierarchy:

| Desktop | Mobile (â‰¤620px) |
| --- | --- |
| `h1`, `.text-h1` | Uses H2 sizing |
| `h2`, `.text-h2` | Uses H3 sizing |
| `h3`, `.text-h3` | Uses H4 sizing |
| `h4`, `.text-h4` | Uses H5 sizing |
| `.text-h5` | Remains H5 sizing |

This ensures optimal typography hierarchy across all screen sizes while maintaining semantic meaning.

### Heading Classes

Instead of relying on downscaled helpers, use explicit heading-level classes to apply typography styles to any element:

| Class | Description | Use Case |
| --- | --- | --- |
| `.text-h1` | Applies H1 styling to any element | When you need H1 visual weight without semantic H1 |
| `.text-h2` | Applies H2 styling to any element | When you need H2 visual weight without semantic H2 |
| `.text-h3` | Applies H3 styling to any element | When you need H3 visual weight without semantic H3 |
| `.text-h4` | Applies H4 styling to any element | When you need H4 visual weight without semantic H4 |
| `.text-h5` | Applies H5 styling to any element | When you need H5 visual weight without semantic H5 |
| `.text-body` | Applies body text styling to any element | For consistent body text on non-text elements |
| `.text-small` | Smaller text size | For captions, footnotes, or secondary information |
| `.text-caption` | Very small text size | For fine print, labels, or minimal details |

## Text

We set the base text styles at `body` and override them as needed. 

The style can be forced to a non _body element_ by using the `.text-body` class.

### Styling Text

#### Text Colors

| Class | Description |
| --- | --- |
| `.text-high` | <span class="text-high">High emphasis text, used where text needs to stand out</span> |
| `.text-low` | <span class="text-low">Low emphasis text, used where text needs to be less prominent</span> |
| `.text-default` | <span class="text-default">Default text color, useful for resetting inherited colors</span> |

#### Text Sizes

| Class | Description |
| --- | --- |
| `.text-h1` | Applies H1 typography styles |
| `.text-h2` | Applies H2 typography styles |
| `.text-h3` | Applies H3 typography styles |
| `.text-h4` | Applies H4 typography styles |
| `.text-h5` | Applies H5 typography styles |
| `.text-body` | Applies body text styles (default) |
| `.text-small` | Smaller text for captions and secondary content |
| `.text-caption` | Even smaller text for fine print and labels |

#### Color Tokens

The text colors are built on semantic color tokens that automatically adapt to light and dark themes:

| Token | Description | Light Mode | Dark Mode |
| --- | --- | --- | --- |
| `--color-text` | Default text color for paragraphs, lists, tables | `var(--chroma-surface-90)` | `var(--chroma-surface-20)` |
| `--color-text-high` | High contrast text for emphasis | `black` | `white` |
| `--color-text-low` | Low contrast text for secondary content | `var(--chroma-surface-80)` | `var(--chroma-surface-40)` |

#### Typography Elements

The following HTML elements automatically use the design system text colors:

- `p` (paragraphs)
- `li` (list items) 
- `td`, `th` (table cells and headers)
- Text within `article`, `section`, and `main` containers

## Tables

Please note: tables are intended for tabular data, not for layout. For layout, we provide a set of grid helpers
and layout containers.

Tables use the token `--color-surface-[+1]` as a highlight color for borders and headers. F.ex `--color-surface-1` for 
the first surface color, `--color-surface-2` on top of `.elevation-1` for the second surface color, etc.
